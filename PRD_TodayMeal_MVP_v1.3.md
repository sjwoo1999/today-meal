# TodayMeal (오늘한끼) PRD
## MVP v1.3 - 심리학 기반 식단 관리 플랫폼

---

# 1. Executive Summary

## 제품 개요
**TodayMeal (오늘한끼)**는 트레이너와 회원을 연결하는 AI 기반 개인화 식단 관리 플랫폼입니다.

> **핵심 슬로건**: "오늘 저녁 뭐 먹지? 그럼 아침엔 이거!"

## 핵심 가치 제안
| 대상 | 문제점 | 솔루션 |
|------|--------|--------|
| **PT 회원** | 식단표를 받아도 실행 어려움, 먹고 싶은 것 포기해야 한다는 박탈감 | 역추산 기능으로 '먹고 싶은 것'을 포기하지 않으면서 목표 달성 |
| **트레이너** | 영양 전문성/리소스 부족, 회원별 관리 시간 과다 | AI 자동 리포트 + 1클릭 피드백 시스템 |

## MVP 핵심 지표 (1년차)
- 파일럿 헬스장: 3-5곳
- MAU: 300-500명
- 주간 기록률: 70%+
- 7일 리텐션: 50%+
- 30일 리텐션: 35%+
- 평균 스트릭: 7일+
- NPS: 50+

## 차별화 전략
**듀오링고 수준의 몰입 경험** + **역추산 플래너** = 지속 가능한 식단 관리

# 2. Problem Statement

## 현재 시장의 Pain Points

### 트레이너 측면
```
1. 영양학/식단 설계 전문성 부족
2. 회원 생활권 주변 메뉴/음식점 정보 파악 어려움  
3. 회원별 식단 기록 수집·분석·피드백 시간 과다
```

### 회원 측면
```
1. 식단표 → 실제 선택 연결 어려움
2. 개인 변수(야근, 알레르기, 예산 등) 미반영
3. 기록 번거로움 + 피드백 지연으로 지속성 저하
4. 먹고 싶은 음식을 포기해야 한다는 스트레스
```

## 심리적 장벽 분석
| 장벽 | 심리적 원인 | 해결 전략 |
|------|-------------|-----------|
| 3일 이상 지속 어려움 | 습관 미형성, 즉각적 보상 부재 | 스트릭 시스템 + 한끼 캐릭터 |
| 기록 귀찮음 | 높은 인지 부하 | 사진 1장 = 기록 완료 |
| 먹고 싶은 것 포기 | 박탈감, 자율성 상실 | 역추산 플래너 |
| 재미없음 | 의무적 느낌 | 게이미피케이션 전면 도입 |

# 7. Feature Specifications with Psychology Mapping

## 7.1 역추산 식단 플래너 (P0) ⭐ 핵심 차별화 기능

### 개요
저녁에 먹고 싶은 메뉴를 선택하면, 하루 목표를 맞추기 위해 아침/점심에 무엇을 먹어야 하는지 역산하여 추천

### 심리학 적용

| 이론 | 적용 방식 |
|------|----------|
| **역방향 계획** | 목표(원하는 저녁)에서 역산하는 계획이 더 효과적 |
| **자기결정이론** | 먹고 싶은 것을 직접 선택 → 자율성 부여 |
| **심적 회계** | 하루 칼로리를 끼니별로 "예산 배분" |
| **손실회피 역이용** | "포기"가 아닌 "허용" 프레이밍 → 박탈감 제거 |

### 상세 플로우

```
Step 1: 저녁 메뉴 검색/선택
├── UI: 검색창 + 인기 메뉴 카드 (삼겹살, 치킨, 피자 등)
├── Input: 메뉴명 또는 음식점명
└── 심리: 선택권 부여로 자율성 충족

Step 2: AI 역산 계산
├── Logic: 일일 목표 - 저녁 예상량 = 아침+점심 예산
├── Variables: 목표 칼로리, 목표 단백질, 저녁 영양정보
└── 심리: 복잡한 계산을 AI가 대신 → 인지부하 감소

Step 3: 아침/점심 추천 제시
├── UI: 타임라인 형태로 하루 식단 시각화
├── Output: 끼니별 추천 + 생활권 내 구매처
└── 심리: 실현 가능한 구체적 행동 제시 → Ability 증가

Step 4: 플랜 확정 & 리마인더
├── UI: "오늘의 플랜 저장" + 알림 설정
├── Gamification: 플랜 세우기 = +10 XP
└── 심리: 공개적 약속으로 실행 확률 증가
```

### Edge Cases

| 상황 | 대응 |
|------|------|
| 저녁 메뉴가 하루 칼로리 80% 이상 | "삼겹살 3인분은 조금 빡빡해 😅 2인분이면 완벽!" |
| 아침을 이미 먹은 후 요청 | 아침 기록 반영하여 점심만 추천 |
| 영양정보 DB에 없는 메뉴 | 유사 메뉴 기반 추정치 + 사용자 피드백 요청 |

### UX Copy 예시
```
"오늘 저녁 뭐 먹고 싶어? 🍽️"
"삼겹살 먹어도 돼! 대신 이렇게 해보자 👇"
"→ 저녁 삼겹살 2인분까지 OK! 🔥"
```

```python
# 역추산 로직 (Pseudo-code)
def calculate_reverse_meal(user, dinner_menu):
    # 입력
    daily_goal = user.daily_calorie_goal
    dinner_nutrition = get_nutrition(dinner_menu)
    already_eaten = user.get_today_records()
    
    # 계산
    remaining_budget = daily_goal - dinner_nutrition - already_eaten
    remaining_meals = get_remaining_meals(user.current_time)
    
    # 제약 조건
    constraints = {
        'min_calories_per_meal': 200,
        'protein_balance': True,
        'user_preferences': user.preferences,
        'location': user.living_area
    }
    
    # 추천 생성
    recommendations = ai_recommend(
        budget=remaining_budget,
        meals=remaining_meals,
        constraints=constraints
    )
    
    return recommendations
```

# 15. Ethical Considerations

## 원칙

| 영역 | 고려사항 | 대응 |
|------|---------|------|
| 손실회피 과용 | 과도한 스트레스 유발 가능 | 프리즈/복구로 완충 |
| 죄책감 유발 | 한끼 멘트가 비난으로 느껴질 수 있음 | 걱정 > 비난 원칙 |
| 과열 경쟁 | 리그 경쟁 스트레스 | 익명 옵션 제공 |
| 알림 피로 | 과도한 푸시 | 하루 최대 3회 제한 |
| 식이장애 | 과도한 제한/집착 유발 가능 | 위험 감지 시 트레이너 알림 |

## 식이장애 위험 대응

```
위험 신호 감지 시:
1. AI가 이상 패턴 감지 (과도한 제한, 폭식/절식 반복)
2. 트레이너에게 알림
3. 사용자에게 전문 리소스 연결 제안
```
